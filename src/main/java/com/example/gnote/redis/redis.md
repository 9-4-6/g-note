
# Redis 三大核心问题
## 一、Redis 雪崩
### 原因
Redis 服务异常（宕机）或大量缓存 **同时失效**，请求直接穿透至 MySQL，导致 MySQL 压垮。
### 解决方案
1.  缓存过期时间 **均匀随机分布**（如 30±5 分钟）
2.  部署 Redis 集群（主从+哨兵/集群模式），避免单点故障

## 二、Redis 击穿
### 原因
**热点 key 缓存失效**，瞬间大量请求穿透至 MySQL。
### 解决方案
1.  热点 key 预缓存（提前加载，延长过期时间）
2.  分布式锁（缓存失效时，串行查询 MySQL 避免并发冲击）

## 三、Redis 穿透
### 原因
请求不存在的 key，Redis 无缓存，所有请求直接打至 MySQL。
### 解决方案
1.  布隆过滤器（提前过滤不存在的 key，拦截无效请求）
2.  空值缓存（不存在的 key 缓存空值+短期过期时间，避免重复穿透）
